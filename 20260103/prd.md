# [PRD] ZPD 디지털 채용 센터 & 수사 본부

(Product Requirements Document)

## 1. 개요 (Introduction)

### 1.1. 프로젝트 배경

- 영화 <주토피아>의 개봉/리부트를 앞두고, 단순 광고를 넘어 사용자가 직접 주토피아의 시민(경찰)이 되는 인터랙티브 경험을 제공하고자 함.
    
- 생성형 AI 기술을 활용한 '나만의 동물 캐릭터 만들기'를 통해 자발적인 SNS 바이럴 유도.
    

### 1.2. 제품 목표 (Goals)

- **Viral:** '주토피아 시민증' 생성 건수 100만 건 달성.
    
- **Conversion:** 참여자 중 20% 이상을 영화 예매 또는 오프라인 팝업 방문으로 유도.
    
- **Retention:** '수사 미션' 기능을 통해 일회성 방문이 아닌 재방문 유도.
    

### 1.3. 주요 타겟

- 모바일 환경에 익숙하고 SNS 인증을 즐기는 2030 세대.
    
- 캐릭터 IP에 호감도가 높은 예비 관객.

## 3. 상세 기능 요구사항 (Functional Requirements)

### 3.1. 온보딩 (Onboarding)

- **[FR-101] 인트로 애니메이션**
    
    - 진입 시 ZPD 로고와 함께 보안 시스템 접속 연출 (Lottie 활용).
        
    - '접속 승인' 텍스트 타이핑 효과 노출.
        
- **[FR-102] 개인정보 수집 동의**
    
    - AI 생성을 위한 사진 업로드 동의 및 마케팅 활용(선택) 동의 팝업.
        
    - **중요:** 업로드된 원본 사진은 AI 생성 후 서버에서 즉시 파기됨을 명시해야 함.
        

### 3.2. AI DNA 스캔 (Core Feature)

- **[FR-201] 이미지 업로드**
    
    - 카메라 촬영 및 갤러리 선택 지원.
        
    - **유효성 검사:** 얼굴이 인식되지 않거나, 다수의 인물이 있는 경우, 유해한 이미지일 경우 에러 메시지 출력 후 재시도 유도.
        
    - _Error Case:_ "DNA 식별 불가. 정면 사진을 사용해 주세요."
        
- **[FR-202] 이미지 프로세싱 (Server-side)**
    
    - 업로드된 이미지를 Vector로 변환하여 주토피아 3D Art Style 모델(Stable Diffusion LoRA 등 튜닝 모델)에 주입.
        
    - 사용자의 성별, 안경 착용 여부, 표정(웃음/무표정) 메타데이터 추출.
        
- **[FR-203] 로딩(Scanning) UI**
    
    - 디자인 명세서 5.1 항목 참조.
        
    - 대기 시간(약 3~5초 예상) 동안 이탈 방지를 위해 'DNA 분석 중...', '잠재적 종(Species) 매칭 중...' 문구 롤링.
        

### 3.3. 결과 페이지 (ZPD 시민증)

- **[FR-301] 카드 렌더링**
    
    - 앞면: 생성된 캐릭터 + 유저 닉네임 + 랜덤 계급(순경/경장/경사) + QR코드(데코).
        
    - 뒷면: 랜덤 부여된 동물 MBTI + 3가지 스탯(Radar Chart).
        
- **[FR-302] 카드 인터랙션**
    
    - 터치 시 3D Flip 효과 적용 (CSS transform).
        
- **[FR-303] 공유 및 저장**
    
    - **이미지 저장:** html2canvas 등을 활용하여 카드 앞/뒷면을 하나의 이미지로 병합하여 다운로드.
        
    - **SNS 공유:** Web Share API 활용, 인스타그램 스토리 공유 시 전용 스티커 오버레이 적용.
        

### 3.4. 수사 본부 (Mission Dashboard)

- **[FR-401] 미션 리스트 관리**
    
    - LocalStorage 또는 로그인 시 DB에 미션 달성 현황 저장.
        
- **[FR-402] GPS 인증 (성수동/스타필드)**
    
    - Geolocation API 활용.
        
    - 지정된 좌표 반경 500m 이내에서 '인증하기' 버튼 클릭 시 성공 처리.
        
    - _Exception:_ 위치 권한 거부 시 권한 요청 가이드 팝업 노출.
        
- **[FR-403] 시크릿 코드 입력**
    
    - 영화 예고편 등에 숨겨진 코드 입력 시 히든 스탬프 획득.
        

## 4. 데이터 요구사항 (Data Requirements)

### 4.1. 수집 데이터

- **User Input:** 닉네임, 업로드 사진(임시), 시크릿 코드.
    
- **System Gen:** User ID (UUID), 생성된 이미지 URL(CDN), MBTI 결과값, 미션 달성 Flag.
    

### 4.2. 데이터 보존 정책

- **원본 사진:** 생성 완료 즉시 영구 삭제 (개인정보 보호 최우선).
    
- **생성된 이미지:** 캠페인 종료일 + 1개월 후 일괄 삭제.
    

## 5. 비기능 요구사항 (Non-Functional Requirements)

### 5.1. 성능 (Performance)

- **LCP (Largest Contentful Paint):** 2.5초 이내.
    
- **AI 생성 시간:** 최대 8초를 넘기지 말 것. 8초 초과 시 "정밀 분석 중"이라는 추가 멘트 노출.
    
- **동시 접속:** 초당 1,000명(TPS) 동시 생성 요청 처리 가능한 GPU 인스턴스 오토스케일링 구성.
    

### 5.2. 호환성 (Compatibility)

- **OS:** iOS 14.0+, Android 8.0+
    
- **Browser:** Chrome, Safari, Samsung Internet, In-app Browser (KakaoTalk, Instagram).
    
    - _Note:_ 인앱 브라우저의 경우 다운로드 기능 제약에 대비해 '링크 복사' 후 외부 브라우저 열기 가이드 제공 필요.
        

### 5.3. 보안 (Security)

- 모든 API 통신은 HTTPS 암호화.
    
- 비정상적인 트래픽(매크로) 감지 시 IP 차단 (Rate Limiting 적용).
